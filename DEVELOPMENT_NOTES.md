# Development Notes

一个本地 Chrome 扩展 demo，可以直接获取当前页面字幕，也可以把字幕和 Prompt 模板发送到 AI 网站。

字幕获取目前支持：

- YouTube：拦截网页 `fetch` / `XMLHttpRequest` 字幕响应，扫描播放器字幕列表，并使用 Performance API 补捞。
- Bilibili：调用官方播放器接口获取字幕列表和内容。
- YouTube/Bilibili 内部 iframe：内容脚本会在对应播放器 frame 中继续捕获字幕。

使用：

1. 在 `chrome://extensions` 更新扩展后，刷新已经打开的视频页面。
2. 字幕模块只会在 YouTube/Bilibili 的具体视频播放页自动显示；频道页、首页和搜索页不会创建字幕框。
3. YouTube 普通模式下模块位于播放器右侧，影院模式下会跟随 YouTube 原生 `#secondary` 布局移动到播放器下方。
4. Bilibili 上模块会嵌入播放器右侧原“弹幕列表”的位置，并监听宽屏模式变化。
5. 模块展开时高度会跟随播放器；顶部箭头可将模块收起为 44px 标题栏。
6. 点击扩展图标可以完全隐藏字幕栏，再点一次可以重新显示。
7. 字幕面板默认显示 `TS`：`CC` 显示逐条带时间戳字幕；`TS` 会把字幕句子合并为较长段落。
8. 没有检测到字幕时面板会自动折叠；检测到字幕后会自动展开。
9. YouTube 必要时开始播放视频或打开一次字幕；Bilibili 会自动查询官方字幕。
10. 点击模块顶部的 `Prompt`，可以使用原有的多套模板和 AI 网站转发功能。

Prompt 转发始终使用字幕下拉框中当前选中的语言轨道，并发送该轨道的 `TS` 分段纯文本；它不读取剪贴板，也不受字幕面板当前显示为 `CC` 还是 `TS` 影响。

`TS` 完全在浏览器本地通过规则生成，不调用 NLTK、云端服务或大模型。每个字幕轨道只生成一次，字幕视图和 Prompt 转发共享同一份缓存结果。

YouTube 模块会等待样式加载完成后再显示，并限制在右栏宽度内；找不到可见且尺寸正常的原生 `#secondary` 时不会退化为全页面面板。

字幕行交互：

- CC 每一行、TS 段落中的每一句都可以定位视频。
- 只有点击或按 Enter/空格才会跳转；鼠标经过不会改变播放进度。
- 视频播放时，CC 当前行或 TS 当前句会实时高亮；正常状态下，当前句离开视窗后列表会自动平滑跟随。
- 手动滚动字幕会暂停滚动跟随，但不会暂停高亮更新；当前句离开视窗后可点击底部的 `Back to Current` 返回并恢复跟随。
- TS 会按字幕语言补充逗号和句号。

支持：

- Google AI Studio
- Gemini
- NotebookLM（辅助模式：复制纯字幕并打开网站）
- ChatGPT
- Claude
- Grok

Prompt 支持多套本地模板，可以新增、重命名、删除，并会记住最后使用的模板。

## 安装

1. 打开 `chrome://extensions`。
2. 打开右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本目录。

## 当前限制

- 没有复刻 vCaptions 的翻译、章节、AI 总结和云端语音转写。
- 某些 Bilibili 字幕要求登录。扩展会使用当前 Bilibili 页面的登录状态请求字幕。
- 使用加密媒体、Canvas 字幕、纯 DOM 实时字幕且没有字幕文件的网站，当前可能无法捕获。
- 网站修改播放器接口或字幕格式后，相应适配器可能需要更新。
- 各网站会不定期修改输入框结构，自动填入规则可能需要更新。
- “自动发送”依赖网站是否接受脚本生成的 Enter 键事件，因此属于实验功能。
- NotebookLM 不是普通聊天页。当前版本会把纯字幕留在剪贴板并打开 NotebookLM，用户需要将其粘贴为 `Copied text` 来源。
- 当前版本每次打开一个新标签页，不会复用已有的 AI 网站标签页。
- 字幕和 Prompt 仅保存在 Chrome 本地存储，并直接填入所选网站。
- 字幕提取器和侧栏目前只注入 YouTube 与 Bilibili；AI Studio、Gemini、ChatGPT 等目标网站只运行文本填入脚本，不显示字幕侧栏。
